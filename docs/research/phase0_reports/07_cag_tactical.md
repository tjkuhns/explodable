# Tactical CAG implementation with Anthropic's prompt caching

**A slot-filling, chain-of-thought, two-pass approach—combined with XML-tagged findings ordered to exploit the U-shaped attention curve—gives the most reliable selection and citation from a 300-finding cached knowledge base.** Here is concrete guidance across all five questions, with clear distinctions between what Anthropic officially documents and what research or practitioner evidence supports.

## 1. Structure the cached system prompt with XML tags and edge-loaded ordering

Anthropic's docs explicitly recommend XML over JSON for document structuring. Use their canonical pattern: wrap each finding in `<finding id="N" category="mechanism" topic="anchoring">` tags nested inside a `<findings>` container. Place a **table of contents** at the very top listing all IDs grouped by category—this falls in the high-attention primacy zone. Put the user's essay prompt in the **user message, not the system prompt**, since Anthropic reports queries at the end improve quality by **up to 30%** on multi-document inputs. Apply `cache_control: {"type": "ephemeral"}` on the final system prompt text block. At ~150 tokens per finding, 300 findings consume ~45K tokens—well within the **200K standard context** and above the 1,024-token caching minimum for Claude Sonnet 4.5. Use automatic caching unless you need separate breakpoints for instructions versus knowledge base content, in which case use two of the four available explicit breakpoints.

## 2. Exploit the U-shaped attention curve, don't fight it

Liu et al. (TACL 2024) demonstrated a **U-shaped performance curve**: retrieval accuracy drops **30%+** for middle-positioned information versus beginning or end. Wu et al. (MIT, 2025) traced this to causal masking and RoPE—it is architectural, not a training artifact, and persists in frontier models. **Place your highest-confidence, most broadly applicable findings in positions 1–50 and 250–300.** Cluster thematically related findings together, but ensure clusters don't all land in the middle. The TOC at the top serves double duty: it occupies the primacy zone and acts as a structural index. Anthropic's own long-context guide confirms: put longform data at the top, ground responses in direct quotes before the main task.

## 3. Use slot-filling plus chain-of-thought in a two-pass architecture

Open-ended selection ("pick the 6 best") is the weakest pattern—it maximizes positional bias. Instead, combine three techniques. **Slot-filling**: "Select 1 primary claim, 3 supporting mechanisms, 2 empirical data points, and 1 counterpoint." This forces scanning across the full context for diverse finding types. **Chain-of-thought**: require Claude to reason about each slot before committing. **Two-pass**: Pass 1 outputs a selection list with IDs and one-sentence justifications; Pass 2 writes the essay using only those findings. Both passes hit the same cached prefix, so the cost overhead is minimal (cache reads cost **0.1× base input price**). Anthropic's own long-context blog found that scratchpad-based quote extraction before answering was the single most impactful technique for accuracy.

## 4. Custom [src:N] markers work but have known fragility at scale

Anthropic's Citations API requires document blocks in **user messages**, not system prompts—so it is incompatible with a cached-system-prompt CAG architecture. Custom `[src:N]` markers remain your best option but are inherently less reliable. Practitioner reports document failure modes: wrong source numbers, missing markers, and fabricated citations, especially for middle-positioned documents. **Mitigation**: the two-pass approach above forces explicit ID commitment before prose generation. Additionally, instruct Claude to quote a key phrase from each finding alongside its `[src:N]` marker—this anchors the citation to actual content and makes hallucinated references detectable in post-processing. There is no published evidence that caching itself changes citation behavior; it affects cost and latency, not model comprehension.

## 5. Build observability around citation distribution and positional bias detection

The Anthropic API exposes `cache_read_input_tokens` and `cache_creation_input_tokens`—monitor these to confirm caching works. For selection quality, **parse `[src:N]` markers from every output and log the distribution across finding positions**. A U-shaped histogram (heavy citation of early and late findings, sparse middle) signals active positional bias. Instrument with **Langfuse** (open-source, self-hostable) to capture prompt versions, token usage, and response content per request. Run periodic **permutation tests**: shuffle finding order and check whether citation patterns shift. For debugging bad selections, enable **extended thinking**—Claude's reasoning trace reveals which findings it considered and rejected. A second "verifier" agent that checks citations against source text catches hallucinated references that slip through.
